This file gives AI assistants context about the project, its conventions, and how to work with it effectively.

## Project Summary

This is a **hardware documentation repository** for an open-source double-conversion (triple-conversion in practice) superheterodyne AM radio receiver. There is no firmware or software source code — the repository contains only:

- Markdown documentation (design description, power analysis, BOM)
- Schematic PDF (EasyEDA export, 12 pages, Rev4 2026-03-28)
- PDF datasheets for key ICs
- Standalone HTML tools (interactive PCB thermal calculator and polygon visualiser)
- MIT License

**There is no build system, no compiler, no test framework, and no firmware to flash.**

## Signal Chain Architecture

The receiver uses a **triple-conversion superheterodyne** topology:

```
SMA Antenna
    │
    ▼
[RF Preselector filter] — LC pi filter, CT301 trimmer tunes centre frequency
    │
    ▼
[BGA2869 LNA — U401] — 32 dB gain, 3.1 dB NF, 5 V, SOT-363
    │
    ▼
[LT5560 Mixer 1 — U501] — CLK0 local oscillator, RF → 10.7 MHz IF
    │
    ▼
[10.7 MHz ceramic BPF — L601]
    │
    ▼
[LT5560 Mixer 2 — U701] — CLK1 local oscillator, 10.7 MHz → 455 kHz IF
    │
    ▼
[455 kHz ceramic BPF — L901]
    │
    ▼
[BGA2869 LNA — U801] — same MMIC, operating at 455 kHz
    │
    ▼
[LT5560 Mixer 3 — U1001] — CLK2 at 455 kHz, synchronous AM detection → audio
    │
    ▼
[LT6202 op-amp — U1101] — differential to single-ended, gain 25.5 (≈28 dB)
    │
    ▼
[PAM8406 class-D amp — U1201] — 2×3 W into 4 Ω from 5 V
    │
    ▼
Speaker (CN1201)
```

Local oscillator frequencies (CLK0, CLK1, CLK2) are generated by the **MS5351M PLL** (U201) locked to a **26 MHz TCXO** (X201) and programmed over **I²C**.

## Power Architecture

Three independent regulated rails from a single DC input (VCC, 6–15 V):

| Rail | Regulator | Package | Max current | Load | Typical current |
|---|---|---|---|---|---|
| +3.3 V | AMS1117-3.3 (U101) | SOT-223 | 800 mA | MS5351M, TCXO | ~34 mA |
| +5 V | AMS1117-5.0 (U104) | SOT-223 | 800 mA | LNAs, mixers, op-amp | ~97 mA |
| +5V_P_AMP | AMS1084CM-5.0 (U102) | TO-263 | 5 A | PAM8406 audio amp only | 100–900 mA |

**Recommended input voltage: 7–9 V DC.** Higher voltages cause more heat in the linear regulators.

## Critical Hardware Facts

These are easy to get wrong and must not be introduced as errors in any documentation update:

### Tab pad net assignments (NOT GND)
All three voltage regulators and all three LT5560 mixers have non-GND exposed pads:

| Component | Exposed pad / tab | Net | PCB action |
|---|---|---|---|
| AMS1117-3.3 (U101) | Tab | **+3.3 V** (Vout) | Isolated polygon on +3.3 V net |
| AMS1084CM-5.0 (U102) | Tab | **+5V_P_AMP** (Vout) | Isolated polygon on +5V_P_AMP net |
| AMS1117-5.0 (U104) | Tab | **+5 V** (Vout) | Isolated polygon on +5 V net |
| LT5560 ×3 (U501, U701, U1001) | EP (pin 9) | **PGND** | Connect to GND plane with vias |

### LT5560 supply voltage limit
- Absolute maximum VCC = **5.3 V**
- The +5 V rail from AMS1117-5.0 is 5.0 V ±2% (4.9–5.1 V), which is within spec under normal conditions
- This limit must not be exceeded; verify during cold start / transients

### BGA2869 (U401, U801)
- SOT-363 package — no exposed thermal pad
- GND pins (2, 4, 5) connect to GND plane with short traces and vias
- Internally matched to 50 Ω — no external matching components needed

### LT5560 third mixer (U1001) topology
- Mixer 3 uses only **one balun** (input side, U1002)
- Output uses **resistive pull-ups** (R1002, R1003 = 200 Ω to +5 V) — not a second balun
- This is the correct low-frequency application per the LT5560 datasheet

## Component Reference Conventions

- Designators follow schematic page numbering: U1xx = power supply, U2xx = clock, U4xx = RF LNA, U5xx = first mixer, etc.
- Capacitors: Cxxx, inductors/filters: Lxxx, resistors: Rxxx, trimmers: CTxxx, diodes: Dxxx, test points: TPxxx, connectors: CNxxx, SMA: RFxxx, crystals: Xxxx
- All ICs sourced from LCSC; part numbers in BOM.md include manufacturer PN and LCSC catalogue number

## Documentation Conventions

- **Language:** All primary documentation is in English. The two HTML tools (`polygon_illustration.html`, `termisk_beregner.html`) are in Danish — do not translate them unless explicitly asked.
- **Units:** SI units throughout. RF values in dB, dBm. Capacitances in pF/nF/µF. Inductances in nH. Currents in mA/A.
- **Tables:** GitHub-flavored Markdown pipe tables with header separator row.
- **Component references** in documentation use the format `U201`, `C101`, `R502`, etc. — always uppercase with no space between letter and number.
- **Net names** in ALL_CAPS_WITH_UNDERSCORES where they appear in schematic context (e.g. `+5V_P_AMP`, `CLK0`, `PGND`, `RF_IN_LNA`).
- **File naming:** UPPERCASE for all Markdown documentation files (README.md, HARDWARE.md, BOM.md, POWER.md).

Two standalone single-file HTML tools are included:

### `termisk_beregner.html` — Thermal Calculator (Danish)
- Interactive tool for AMS1084CM-5.0 thermal design
- Inputs: VCC (6–15 V), audio current (50–900 mA), copper layer count
- Outputs: power dissipated, required thermal resistance, required copper area, pass/fail verdict

- Shows via placement, copper area distribution, thermal relief design

## What AI Assistants Should and Should Not Do

### Safe to do
- Edit or extend Markdown documentation (README, HARDWARE, POWER, BOM)
- Correct factual errors in documentation against the source files and datasheets
- Add new Markdown files for additional analysis, errata, or assembly notes
- Update BOM.md with revised part numbers or LCSC availability
- Explain circuit operation, component choices, or design trade-offs based on the existing docs
- Answer questions about PCB layout using the guidelines in HARDWARE.md, POWER.md, and BOM.md

### Require caution / ask first
- Changing component values, designators, or part numbers in BOM.md
- Modifying the signal chain description in HARDWARE.md
- Altering the tab-pad net assignments or thermal recommendations
- Updating schematic references (pages, pin numbers, net names)

### Do not do
- Create or modify the HTML tools (they are complete and language-specific)
- Add firmware, driver, or software code unless the project scope changes
- Infer or invent schematic details — the schematic PDF is now available; use it as ground truth
- Silently "fix" the BOM designator discrepancies listed above without confirming with the owner
